# 📲 订阅消息功能配置指南

## 概述

本功能使用微信小程序的**订阅消息**能力，在日程提醒时间到达时向用户发送通知。

---

## 一、获取订阅消息模板

### 步骤 1：登录微信公众平台

访问 https://mp.weixin.qq.com 并登录你的小程序账号。

---

### 步骤 2：进入订阅消息配置

1. 点击左侧菜单 **「订阅消息」**
2. 点击 **「公共模板库」**

---

### 步骤 3：添加模板

在公共模板库中搜索关键词，找到合适的模板。

**推荐模板关键词：**
- `日程提醒`
- `日程通知`
- `时间提醒`

找到后点击 **「添加」** 按钮。

---

### 步骤 4：获取模板ID

添加成功后，在 **「我的模板」** 中可以看到：

```
模板ID: xxxxxxxxxxxxxxxxxxxxxxxx
模板标题: 日程提醒
```

**复制这个模板ID，后续配置需要用到。**

---

## 二、配置代码

### 步骤 1：更新云函数中的模板ID

找到云函数 `cloudfunctions/send-reminder/index.js`，修改第 11 行：

```javascript
// 原来的
const TEMPLATE_ID = 'YOUR_TEMPLATE_ID';

// 修改为你的模板ID，例如：
const TEMPLATE_ID = 'abcdefghijk1234567890';
```

同样更新 `cloudfunctions/reminder-trigger/index.js` 中的模板ID。

---

### 步骤 2：更新小程序配置

找到 `miniprogram/pages/schedule/schedule.js`，在 `data` 中添加：

```javascript
data: {
  // ... 其他配置
  
  // 订阅消息相关
  reminderEnabled: true,      // 是否启用提醒
  reminderTemplateId: '你的模板ID',  // 订阅消息模板ID
}
```

---

## 三、部署云函数

### 部署步骤

1. 打开 **微信开发者工具**
2. 找到 `cloudfunctions/send-reminder` 文件夹
3. **右键点击** → **「上传并部署：云端安装依赖」**
4. 等待部署完成（显示绿色 ✓）

同样方式部署：
- `cloudfunctions/reminder-trigger`
- `cloudfunctions/reminder`

---

### 配置定时触发器（可选）

如果你希望自动发送提醒，需要配置定时触发器：

1. 在微信开发者工具中，点击 **「云开发」**
2. 进入 **「云函数」**
3. 点击 `reminder-trigger` 函数
4. 选择 **「定时触发器」** 标签
5. 点击 **「添加触发器」**

**触发周期设置：**
- 推荐：`0 */5 * * * *` （每5分钟执行一次）
- 或者：`0 0 * * * *` （每小时执行一次）

---

## 四、创建数据库集合

### 创建 reminders 集合

1. 在云开发控制台，点击 **「数据库」**
2. 点击 **「添加集合」**
3. 输入集合名称：`reminders`
4. 设置权限为 **「仅创建者可读写」**

**集合字段说明：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| _id | string | 自动生成 |
| schedule_id | string | 关联的日程ID |
| reminder_time | date | 提醒时间 |
| status | string | pending/sent/failed/cancelled |
| created_at | date | 创建时间 |
| sent_at | date | 发送时间 |

## 六、常见问题

### Q1: 用户没有收到提醒

**可能原因：**
1. 用户未授权订阅消息
2. 模板ID配置错误
3. 云函数未正确部署
4. 定时触发器未配置

**解决方法：**
1. 检查用户在创建日程时是否点击了「允许」
2. 确认云函数中的模板ID正确
3. 查看云函数日志排查错误

---

### Q2: 提示"模板ID无效"

**解决方法：**
1. 确认模板ID正确无误
2. 确认模板已在小程序后台添加
3. 检查模板是否与小程序账号匹配

---

### Q3: 订阅消息弹窗不显示

**可能原因：**
1. 用户之前已拒绝授权
2. 小程序未完成认证

**解决方法：**
1. 提示用户：设置 → 小程序 → 找到你的小程序 → 打开通知
2. 或引导用户重新进入小程序再次触发

---

### Q4: 如何让用户多次收到提醒？

微信订阅消息有以下限制：
- 一次性订阅：用户同意后只能发送一次
- 长期订阅：用户同意后可长期发送（需要特定模板）

**解决方案：**
1. 使用 **一次性订阅**：每次创建日程时引导用户授权
2. 使用 **长期订阅**（推荐）：在模板库中找到支持长期订阅的模板

## 七、用户体验优化建议

### 1. 清晰的授权提示

在用户点击开启提醒时，显示友好的说明：

```
📲 开启提醒
开启后，我们会在「日程开始前」通过微信通知您
您可以随时在设置中关闭此功能
```

---

### 2. 提醒偏好设置

在个人中心添加提醒设置选项：

- 默认提醒时间
- 是否开启声音
- 是否开启震动

---

### 3. 提醒历史记录

允许用户查看历史提醒记录，了解哪些日程已发送提醒。

---

## 八、技术架构图

```
用户创建日程
    ↓
前端请求订阅授权
    ↓
用户点击「允许」
    ↓
保存日程 + 创建提醒记录
    ↓
定时任务（每5分钟）
    ↓
查询待发送提醒
    ↓
调用订阅消息API
    ↓
用户收到微信通知
```
